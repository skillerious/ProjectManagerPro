How Key Generation Works

Understanding the license key generation process and output format.

Generation Process

Keys are generated using a cryptographically secure HMAC-SHA256 algorithm. A secret seed is combined with random entropy and the selected license tier to produce unique, non-guessable license keys.

Each key embeds a tier identifier and checksum, allowing the main application to validate keys entirely offline without any network requests.

Output Format ####-####-####-#### 16 digits in 4 groups
Batch Size 1 – 200 Keys per generation
Algorithm HMAC-SHA256 With embedded tier data
Validation Offline No network required

Activating in the Main Application

How to use generated keys to register Project Manager Pro.

1

Generate & Copy a Key

Select your desired license tier, set the batch size, and click Generate Keys. Click any key row to copy a single key, or use Copy All to copy the entire batch to your clipboard.

2

Open the Registration Dialog

In Project Manager Pro, click on the registration status in the bottom status bar, or navigate to Menu → Register Product to open the activation dialog.

3

Paste & Register

Paste the copied key into the product key input field. The key format is validated automatically as you type. Click Register to activate. Validation is performed locally using HMAC verification.

4

Activation Complete

Activation is instant and fully offline. The license is bound to the current device using a hardware fingerprint. If hardware changes are detected, a 7-day grace period allows continued use while you re-register.

Security & Validation

How keys are validated, stored, and bound to devices.

Offline Validation

All key validation is performed entirely offline. No network requests are made during the activation process. Keys are verified using HMAC-SHA256 with an embedded checksum and tier identifier, ensuring tamper-proof validation.

Hardware Binding

Upon activation, the license is bound to the device using a fingerprint derived from the CPU identifier, operating system identity, and current user profile. This prevents unauthorized key sharing across machines.

Key Storage Encrypted User app data directory
Device Binding Fingerprint CPU + OS + User
Grace Period 7 Days After hardware change
Network None Fully offline activation
